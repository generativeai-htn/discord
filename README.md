# สคริปต์สร้างห้อง Discord อัตโนมัติ — วิทยาลัยเทคโนโลยีไฮเทค หนองไผ่ (HTN)

สคริปต์นี้จะสร้างหมวดหมู่และห้องต่างๆ ในเซิร์ฟเวอร์ Discord ของวิทยาลัย
ตามโครงสร้างที่ออกแบบไว้ (ดูรายละเอียดในไฟล์ setup-server.js)

## ขั้นตอนการใช้งาน

### 1. สร้างบอท Discord
1. ไปที่ https://discord.com/developers/applications
2. กด **New Application** ตั้งชื่อบอท
3. ไปที่แท็บ **Bot** → กด **Reset Token** เพื่อคัดลอกโทเคนบอท (เก็บเป็นความลับ ห้ามแชร์ให้ใคร)
4. เปิดสิทธิ์ **Privileged Gateway Intents** ตามต้องการ (ไม่จำเป็นสำหรับสคริปต์นี้)

### 2. เชิญบอทเข้าเซิร์ฟเวอร์
1. ไปที่แท็บ **OAuth2 > URL Generator**
2. เลือก scope: `bot`
3. เลือก permission: `Manage Channels`
4. คัดลอกลิงก์ที่ได้ แล้วเปิดในเบราว์เซอร์เพื่อเชิญบอทเข้าเซิร์ฟเวอร์ใหม่ (หรือเซิร์ฟเวอร์ที่ต้องการสร้างห้อง)

### 3. ตั้งค่าไฟล์ .env
สร้างไฟล์ชื่อ `.env` ในโฟลเดอร์เดียวกับ setup-server.js แล้วใส่:

```
BOT_TOKEN=โทเคนบอทของคุณ
GUILD_ID=ไอดีเซิร์ฟเวอร์ของคุณ
```

วิธีหา GUILD_ID: เปิด Discord → Settings → Advanced → เปิด Developer Mode
แล้วคลิกขวาที่ชื่อเซิร์ฟเวอร์ → Copy Server ID

### 4. ติดตั้งและรันสคริปต์

```bash
npm install
node setup-server.js       # สร้างหมวดหมู่/ห้องทั้งหมด
node cleanup-defaults.js   # ลบหมวดหมู่เริ่มต้นของ Discord ที่ไม่ใช้ (Information/Text/Voice Channels)
node setup-roles.js        # สร้าง role ประจำสาขา + role ยืนยันตัวตน และจำกัดสิทธิ์ห้อง
node setup-classes.js      # สร้าง role ห้องเรียน (ปวช.1/1 ฯลฯ) — /verify จะให้อัตโนมัติตามห้องในไฟล์
```

**role ห้องเรียน:** เมื่อนักเรียนยืนยันตัวตน บอทให้ role ห้อง (เช่น ปวช.1/1) อัตโนมัติตามข้อมูลในไฟล์
ครูจึง @mention ทั้งห้องได้ เช่น "@ปวช.1/1 พรุ่งนี้มีสอบ"

**ปีการศึกษาใหม่ / อัปเดตรายชื่อ:** แทนที่ไฟล์ `รายชื่อนักศึกษา.xlsx` แล้วพิมพ์ `/reload-roster` (แอดมิน)
บอทจะโหลดรายชื่อใหม่ + สร้าง role ห้องใหม่ให้อัตโนมัติ **โดยไม่ต้องรีสตาร์ท**

ทุกสคริปต์เป็น idempotent รันซ้ำได้ปลอดภัย จะข้ามห้อง/role ที่มีอยู่แล้ว
เหมาะกับตอนเพิ่มสาขาใหม่ภายหลัง (แก้ไขในไฟล์ `branches.js` แล้วรัน `setup-server.js` + `setup-roles.js` ใหม่)

**หมายเหตุสิทธิ์บอท:** `setup-roles.js` ต้องการสิทธิ์ **"จัดการบทบาท" (Manage Roles)** เพิ่มจาก
"จัดการแชนแนล" ที่ให้ไว้ตอนแรก (ตั้งได้ที่ Server Settings → Roles → htn_bot)

### 5. ยืนยันตัวตนนักเรียน (/verify)

วางไฟล์ `รายชื่อนักศึกษา.xlsx` ไว้ในโฟลเดอร์เดียวกัน (คอลัมน์ที่ใช้: เลขประจำตัวนักเรียน,
ห้องเรียน, ชื่อ-นามสกุล, สถานะนักเรียน — **ไม่อ่านเลขบัตรประชาชนหรือวันเกิด**) แล้วรันบอทค้างไว้:

```bash
node bot.js
```

นักเรียนกดปุ่ม "✅ ยืนยันตัวตนนักเรียน" ในห้อง #รับยศเข้าใช้งาน (หรือพิมพ์คำสั่ง `/verify` ก็ได้เหมือนกัน)
กรอกเลขประจำตัวนักเรียน + ชื่อ-นามสกุลในฟอร์มที่เปิดขึ้นมา ถ้าตรงกับข้อมูลในไฟล์ บอทจะตั้งชื่อเล่นเป็นชื่อจริง
และให้ role ประจำสาขา + role "นักเรียน (ยืนยันแล้ว)"

### 6. ถามตอบกับ AI (/ask)

ใส่ค่า `OPENROUTER_API_KEY` ในไฟล์ `.env` (สมัครฟรีได้ที่ https://openrouter.ai/keys)
บอทตัวเดียวกัน (`bot.js`) จะเปิดคำสั่ง `/ask` ให้ถามคำถามเกี่ยวกับวิทยาลัยได้ทันที
ตอบแบบ ephemeral (เห็นเฉพาะผู้ถาม) ปรับ system prompt/ขอบเขตคำตอบได้ที่ไฟล์ `ai-config.js`
เปลี่ยนโมเดลได้ที่ตัวแปร `OPENROUTER_MODEL` ในไฟล์เดียวกัน (ค่าเริ่มต้นใช้โมเดลฟรี ต่อท้ายด้วย `:free`)

### 7. เจนภาพด้วย AI (/image)

ใช้ Pollinations.ai ฟรี ไม่ต้องมี API key ใดๆ เพิ่ม พิมพ์คำสั่ง `/image รายละเอียดภาพ: ...`
เพื่อสร้างภาพสำหรับโปสเตอร์/ตกแต่งเซิร์ฟเวอร์ (ไม่มีปุ่มให้กด ใช้พิมพ์คำสั่งเท่านั้น)
**จำกัดสิทธิ์เฉพาะคนมี "Manage Server"** โดยค่าเริ่มต้น
(ปรับสิทธิ์ให้ role อื่น เช่น "ทีมงานประชาสัมพันธ์" ใช้ได้ที่ Server Settings → Integrations → htn_bot →
Command permissions โดยไม่ต้องให้สิทธิ์ Manage Server เต็มๆ)

### 8. โพสต์กฎระเบียบ + คู่มือ + ปุ่มโต้ตอบ (setup-content.js)

```bash
node setup-content.js
```

โพสต์กฎระเบียบลงห้อง #กฎ-ระเบียบ, คู่มือการใช้งานพร้อมปุ่ม "ยืนยันตัวตน/ถาม AI" ลงห้อง
#คู่มือการใช้งานเซิร์ฟเวอร์ และปุ่มยืนยันตัวตนลงห้อง #รับยศเข้าใช้งาน ทำให้สมาชิกใช้งานฟีเจอร์หลักผ่านการกดปุ่มได้เลย
ไม่ต้องพิมพ์คำสั่งเอง (คำสั่ง /verify /ask /image ยังใช้งานได้คู่ขนานกันเสมอ)
เป็นแบบ "upsert" รันซ้ำได้เรื่อยๆ — ถ้าแก้เนื้อหาในไฟล์แล้วรันใหม่ จะแก้ข้อความเดิมให้ตรงกับล่าสุดเสมอ

### 9. ระบบตรวจข้อความผิดกฎอัตโนมัติ (setup-modlog.js)

```bash
node setup-modlog.js
```

สร้างห้อง **#mod-log** (ซ่อนจาก @everyone โดยตั้งใจ เจ้าของ/แอดมินเห็นอัตโนมัติ ถ้าต้องการให้ทีมงาน
คนอื่นเห็นด้วยต้องไปเพิ่มสิทธิ์เองที่ตั้งค่าห้อง) สำหรับเก็บประวัติข้อความที่ถูกลบ

ระบบตรวจข้อความทุกข้อความในเซิร์ฟเวอร์ 2 ชั้น (`moderation.js`):
1. **บัญชีคำต้องห้าม** (`banned-words.js` → `HARD_BLOCKLIST`) — คำหยาบชัดเจน ลบทันที ไม่ต้องเรียก AI
2. **บัญชีคำเฝ้าระวัง** (`WATCHLIST`) — คำที่อาจล่อแหลม/สุ่มเสี่ยงแต่กำกวม ส่งให้ AI ช่วยตัดสินอีกชั้น

เจอข้อความผิดกฎ: ลบข้อความ + เตือนสมาชิกในห้อง (ข้อความเตือนจะหายไปเองใน 8 วิ) + บันทึกจำนวนครั้งที่เตือน
ไว้ในไฟล์ `warnings.json` + ส่งรายละเอียดเต็มไปที่ #mod-log ให้แอดมินตรวจสอบ

**ลงโทษอัตโนมัติตามจำนวนเตือน:** ครบ 3 ครั้ง → Timeout 10 นาที · 5 ครั้ง → 1 ชม. · 7 ครั้งขึ้นไป → 24 ชม.
(ต้องมีสิทธิ์ "หมดเวลาสมาชิก/Timeout Members" ซึ่งเปิดไว้แล้ว — บังคับกับแอดมิน/เจ้าของไม่ได้ตามข้อจำกัด Discord)

**เพิ่ม/ลบคำต้องห้ามได้เองตอนรัน (ไม่ต้องแก้โค้ด/รีสตาร์ท):**
- `/badword-add คำ ประเภท` — เพิ่มคำ (บล็อกทันที / เฝ้าระวังให้ AI ดู) มีผลทันที เก็บใน `custom-words.json`
- `/badword-remove คำ` · `/badword-list` — ลบ/ดูคำที่เพิ่มเอง (บัญชีในโค้ด banned-words.js ยังทำงานเสมอ)

**⚠️ ต้องเปิด "MESSAGE CONTENT INTENT" ก่อนใช้งาน** ที่ https://discord.com/developers/applications
→ เลือกแอปบอท → แท็บ **Bot** → เลื่อนหา **Privileged Gateway Intents** → เปิด **Message Content Intent**
→ Save Changes (ไม่งั้นบอทจะอ่านเนื้อหาข้อความไม่ได้ ระบบตรวจจะไม่ทำงานเลย)

เพิ่ม/ลบคำในบัญชีได้ที่ไฟล์ `banned-words.js` ตามการใช้งานจริงของเซิร์ฟเวอร์

### 10. ระบบต้อนรับสมาชิกใหม่ (setup-welcome.js)

```bash
node setup-welcome.js   # สร้างห้อง #ยินดีต้อนรับ พร้อมป้ายต้อนรับ+ปุ่มลัด
```

พอมีสมาชิกใหม่เข้าเซิร์ฟเวอร์ บอทจะโพสต์ต้อนรับใน #ยินดีต้อนรับ อัตโนมัติ พร้อมชี้ทางไปยืนยันตัวตน
**⚠️ ต้องเปิด "SERVER MEMBERS INTENT"** ใน Developer Portal → Bot → Privileged Gateway Intents ด้วย
(แบบเดียวกับ Message Content Intent) ไม่งั้นบอทจะล็อกอินไม่ผ่าน

ต้องการใส่ภาพประกอบ/แบนเนอร์เอง: อัปโหลดภาพในห้องนั้นได้โดยตรง หรือใช้คำสั่ง `/image` ให้ AI เจนภาพ

### 11. ระบบประกาศตามตาราง (/announce)

- `/announce ห้อง เวลา ข้อความ` — ตั้งประกาศล่วงหน้า (เวลารูปแบบ `YYYY-MM-DD HH:mm`) บอทโพสต์เองเมื่อถึงเวลา
- `/announce-list` — ดูรายการประกาศที่รอโพสต์
- `/announce-cancel หมายเลข` — ยกเลิกประกาศ

เฉพาะคนมีสิทธิ์ Manage Server ข้อมูลเก็บใน `announcements.json` อยู่รอดผ่านการรีสตาร์ทบอท

### 12. ระบบครู (setup-teachers.js)

```bash
node setup-teachers.js   # สร้าง role คุณครู + ห้องพักครู + ปุ่มยืนยันตัวตนครู
```

วางไฟล์ `รายชื่อครู.xlsx` (คอลัมน์: ชื่อ-นามสกุล | ตำแหน่ง | ...) ไว้ในโฟลเดอร์ — ระบบอ่านเฉพาะชื่อ+ตำแหน่ง
ครูกดปุ่ม "🍎 ยืนยันตัวตนครู" กรอกชื่อ → คำขอส่งเข้า #mod-log → **แอดมินกดอนุมัติ** ถึงได้ role
role คุณครูได้สิทธิ์: เห็นห้องทุกสาขา, เข้าห้องพักครู, ลบข้อความ/timeout, ใช้คำสั่งประกาศ/จัดแข่งได้
**⚠️ ต้องเปิดสิทธิ์บอท "หมดเวลาสมาชิก (Timeout Members)" ก่อน** ไม่งั้นสร้าง role ไม่ได้

ป้องกันการแอบอ้าง 4 ชั้น: แอดมินอนุมัติ + จับโป๊ะบัญชีที่ยืนยันเป็นนักเรียนแล้ว/บัญชีใหม่ (การ์ดแดงเตือน)
+ ล็อก 1 ชื่อ = 1 บัญชี (`teacher-claimed.json`) + กันซ้ำตอนอนุมัติ

### 13. ระบบศิษย์เก่า (setup-alumni.js)

```bash
node setup-alumni.js   # สร้าง role ศิษย์เก่า + ห้อง #ศิษย์เก่า + ห้องลับ #ข้อมูลศิษย์เก่า + ปุ่มลงทะเบียน
```

ศิษย์เก่ากดปุ่ม "🎓 ลงทะเบียนศิษย์เก่า" กรอกข้อมูล (ชื่อ, ปีที่จบ/สาขา, สถานะ, ที่ทำงาน, เบอร์/อีเมล)
→ ได้ role ศิษย์เก่าทันที (self-service) เข้าห้อง #ศิษย์เก่า ได้
**ความเป็นส่วนตัว (PDPA):** ข้อมูลส่วนตัวส่งเข้าห้อง #ข้อมูลศิษย์เก่า ที่เห็นเฉพาะแอดมินเท่านั้น
ไม่โพสต์สาธารณะ ไม่เก็บเป็นไฟล์ (ลดสำเนาข้อมูลส่วนตัว)

### 14. ระบบสนุกๆ สำหรับคอมมูเกม (setup-fun.js)

```bash
node setup-fun.js   # สร้าง role เลเวล + role เกม + แผงปุ่มเลือกเกมในห้อง #รับยศเข้าใช้งาน
```

- **เลเวล/XP** — พิมพ์คุยได้ XP+เหรียญอัตโนมัติ (คูลดาวน์กันสแปม) เลเวลอัปได้ role ตามเลเวล (🌱 Lv.5 ... 👑 Lv.50)
  - `/rank [ผู้ใช้]` ดูเลเวล/XP + แถบความคืบหน้า · `/leaderboard` อันดับสูงสุด
- **เหรียญ + เกมเสี่ยงโชค** (เงินปลอมในเกม) — `/daily` รับรายวัน · `/balance` ดูเหรียญ · `/give` โอนให้เพื่อน
  - `/coinflip เดิมพัน ทาย` · `/slot เดิมพัน` · `/dice เดิมพัน`
- **Reaction Role เกม** — กดปุ่มเลือกเกมที่เล่น (Roblox/FiveM/ROV/Valorant/Minecraft ฯลฯ) รับ role เพื่อหาเพื่อนเล่น
  - แก้ไขรายชื่อเกมได้ที่ `game-roles.js`
- **Quiz** — `/quiz คำถาม เฉลย [รางวัล]` (ครู/แอดมิน) โพสต์คำถาม ใครพิมพ์ตอบถูกเร็วสุดในห้องได้เหรียญ (หมดเวลา 60 วิ)

ข้อมูล XP/เหรียญเก็บใน `economy.json` **ต้องเปิด Message Content Intent อยู่แล้ว** (ใช้ตัวเดียวกับระบบตรวจคำหยาบ)

### 15. เว็บแดชบอร์ดหลังบ้าน (dashboard.js)

แดชบอร์ดคุมบอทจากเว็บ ไม่ต้องพิมพ์คำสั่งใน Discord — รันอยู่ในโปรเซสเดียวกับบอท (`node bot.js`)

ตั้งค่าใน `.env`:
```
DASHBOARD_PORT=3000
DASHBOARD_PASSWORD=<รหัสผ่านที่แข็งแรง>
```
ถ้าไม่ตั้ง `DASHBOARD_PASSWORD` แดชบอร์ดจะไม่เปิด (เพื่อความปลอดภัย) เปิดใช้แล้วเข้าที่ `http://localhost:3000`
ล็อกอินด้วยรหัสผ่าน ครอบคลุม 4 ส่วน:
- **📊 สถิติ** — จำนวนสมาชิก/นักเรียน/ครู/ศิษย์เก่า, อันดับ XP, ประกาศรอส่ง
- **📢 ประกาศ** — ส่งข้อความเข้าห้องทันที หรือตั้งเวลาล่วงหน้า + ดู/ยกเลิกประกาศที่รอ
- **👥 สมาชิก/Role** — ค้นหาสมาชิก เพิ่ม/ถอด role
- **🏆 จัดแข่ง** — เปิดการแข่งขัน (โพสต์ปุ่มรับสมัครใน Discord), ปิด/เปิดรับสมัคร, จับสู่

**⚠️ ความปลอดภัย:** แดชบอร์ดสั่งงานบอทได้เต็มที่ ตั้งรหัสผ่านที่แข็งแรงเสมอ ถ้าเปิดออกอินเทอร์เน็ต
ควรวางหลัง reverse proxy + HTTPS หรือจำกัดเข้าเฉพาะ LAN/VPN (ค่าเริ่มต้น bind 0.0.0.0 ต้องมีรหัสผ่าน)

**บอทต้องรันค้างไว้ตลอดเวลา** ถ้าปิดเทอร์มินัลหรือรีสตาร์ทเครื่อง ทั้งปุ่ม คำสั่ง ระบบตรวจข้อความผิดกฎ
ระบบต้อนรับ ประกาศตามตาราง และแดชบอร์ด จะใช้งานไม่ได้จนกว่าจะรัน `node bot.js` ใหม่ — ถ้าต้องการให้รันถาวรแนะนำใช้ตัวจัดการโปรเซส เช่น `pm2`

## โครงสร้างสาขาที่ตั้งไว้ล่วงหน้า

- เทคโนโลยีธุรกิจดิจิทัล-คอมพิวเตอร์ธุรกิจ
